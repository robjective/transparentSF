Creating Evaluations

Use the buttons above to create new eval groups and individual evals. Eval groups help organize related tests, while individual evals define specific test cases.

Eval Groups

  • Name: A descriptive name for the group
  • Category: The type of functionality being tested (metrics, system, features)
  • Description: Detailed description of what the group tests

Evals

  • Prompt: The test prompt to send to the AI
  • Success Criteria: How to determine if the test passed
  • Success Type: binary (pass/fail), scored (0-1), or multi_step
  • Expected Tool Calls: Which functions the AI should call
